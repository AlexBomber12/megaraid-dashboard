## Problem

A new operator deploying megaraid-dashboard from scratch currently follows an ad-hoc sequence of manual commands tracked only in the original author's memory. The roadmap §2.8 commits: "A new operator deploys from `INSTALL.md` alone." That requires a runnable `scripts/install.sh` that codifies the steps, plus a companion `INSTALL.md` (PR-056). This PR ships the FIRST OF FOUR install.sh stages: pre-flight environment checks plus system user / directory creation.

Splitting install.sh into four PRs (PR-028, PR-029, PR-030, PR-031) keeps each PR under 200 lines and lets each stage be reviewed and tested independently.

## Scope

In scope:
- New `scripts/install.sh` skeleton with `set -euo pipefail`, top-of-file usage doc, and a `main` entry point that calls phase functions.
- Phase 1: pre-flight checks. Verify Ubuntu 24.04 (`/etc/os-release`), Python 3.12 (`python3 --version`), `storcli64` present at `/usr/local/sbin/storcli64`, port 8090 free (`ss -lnt | grep 8090`), egress to `pypi.org` reachable (one curl).
- Phase 2: create system user `megaraid` (or whatever the unit file declares; coordinate with PR-012's deploy/megaraid-dashboard.service), home `/var/lib/megaraid-dashboard`, no shell.
- Phase 3: create directories: `/opt/megaraid-dashboard` (root-owned, group-megaraid, mode 0750), `/var/lib/megaraid-dashboard` (megaraid-owned, mode 0750), `/etc/megaraid-dashboard` (root-owned, mode 0750), `/etc/megaraid-dashboard/env` placeholder file (mode 0600).
- Idempotency: every step checks if the resource already exists and skips, with a clear log line.
- Helper bash functions: `log_info`, `log_warn`, `log_fail`, `require_root`, `command_exists`.
- Tests using shellcheck and a containerized smoke test (Docker-from-Podman or just shellcheck for now; full integration test deferred to PR-031).

Out of scope:
- venv creation / pip install. PR-029.
- Config wizard (interactive prompts). PR-030.
- sudoers fragment / systemd unit / final smoke. PR-031.
- Companion `scripts/uninstall.sh`. PR-031.
- INSTALL.md prose. PR-056 (PR-DOCS-01a).

## Files to create

`scripts/install.sh` (skeleton plus phase 1, 2, 3):

```bash
#!/usr/bin/env bash
# Megaraid Dashboard installer.
# Usage: sudo bash scripts/install.sh [--non-interactive]
set -euo pipefail

INSTALL_USER="megaraid"
INSTALL_PREFIX="/opt/megaraid-dashboard"
DATA_DIR="/var/lib/megaraid-dashboard"
ETC_DIR="/etc/megaraid-dashboard"
ENV_FILE="${ETC_DIR}/env"
STORCLI_PATH="/usr/local/sbin/storcli64"
APP_PORT="8090"

log_info() { printf "\033[1;34m[info]\033[0m %s\n" "$*"; }
log_warn() { printf "\033[1;33m[warn]\033[0m %s\n" "$*" >&2; }
log_fail() { printf "\033[1;31m[fail]\033[0m %s\n" "$*" >&2; exit 1; }

require_root() {
  [[ ${EUID} -eq 0 ]] || log_fail "must run as root"
}
command_exists() { command -v "$1" >/dev/null 2>&1; }

phase_preflight() {
  log_info "Phase 1: pre-flight"
  if ! grep -q '^ID=ubuntu' /etc/os-release; then
    log_fail "expected Ubuntu, got $(grep '^ID=' /etc/os-release || echo unknown)"
  fi
  local ver
  ver="$(grep '^VERSION_ID=' /etc/os-release | cut -d'"' -f2)"
  [[ "${ver}" == "24.04" ]] || log_warn "expected Ubuntu 24.04, got ${ver}"
  command_exists python3 || log_fail "python3 not found"
  local py_ver
  py_ver="$(python3 -c 'import sys; print(f"{sys.version_info.major}.{sys.version_info.minor}")')"
  [[ "${py_ver}" == "3.12" ]] || log_warn "expected Python 3.12, got ${py_ver}"
  [[ -x "${STORCLI_PATH}" ]] || log_fail "storcli64 not found at ${STORCLI_PATH}"
  if ss -lnt | awk '{print $4}' | grep -q ":${APP_PORT}\$"; then
    log_fail "port ${APP_PORT} already in use"
  fi
  log_info "pre-flight OK"
}

phase_user() {
  log_info "Phase 2: system user"
  if id -u "${INSTALL_USER}" >/dev/null 2>&1; then
    log_info "user ${INSTALL_USER} already exists, skip"
  else
    useradd --system --home-dir "${DATA_DIR}" --no-create-home --shell /usr/sbin/nologin "${INSTALL_USER}"
    log_info "created user ${INSTALL_USER}"
  fi
}

phase_dirs() {
  log_info "Phase 3: directories"
  install -d -m 0750 -o root -g "${INSTALL_USER}" "${INSTALL_PREFIX}"
  install -d -m 0750 -o "${INSTALL_USER}" -g "${INSTALL_USER}" "${DATA_DIR}"
  install -d -m 0750 -o root -g "${INSTALL_USER}" "${ETC_DIR}"
  if [[ ! -f "${ENV_FILE}" ]]; then
    install -m 0600 -o root -g "${INSTALL_USER}" /dev/null "${ENV_FILE}"
    log_info "created ${ENV_FILE} (empty placeholder)"
  else
    log_info "${ENV_FILE} exists, skip"
  fi
}

main() {
  require_root
  phase_preflight
  phase_user
  phase_dirs
  log_info "PR-028 phases complete; continue with venv / pip / config / systemd via PR-029..PR-031"
}

main "$@"
```

## Implementation notes

The installer is intentionally non-interactive in this PR: PR-030 adds the wizard. This phase only runs deterministic checks and idempotent file ops, so re-running it is safe.

`install -d` is the right tool for "create dir if not exists with these owner/mode"; do not `mkdir -p` plus `chmod`/`chown`: `install` is one syscall and atomic.

The system user has no login shell (`/usr/sbin/nologin`) and no home creation. systemd will start the service as this user; nobody logs in as it.

Mode 0750 on the directories restricts world access. The env file is mode 0600 (owner-only) because it will hold SMTP credentials and the bcrypt hash.

The pre-flight checks are best-effort. Some are warnings, not failures (e.g. wrong Ubuntu point release). The hard fails are: not Ubuntu, no python3, no storcli64, port in use, not root.

The egress check is left as a TODO; the PR-029 venv/pip phase will need pypi reachable, but at this stage we have not yet hit the network. Shifting the network check to PR-029 keeps phase 1 scope minimal.

shellcheck is the bash equivalent of ruff; treat as mandatory if installed, warning if absent. CI should install it (apt-get install shellcheck).

## Success criteria

1. `python -m ruff check .`, `ruff format --check .`, `mypy src`, `pytest` all pass.
2. `bash -n scripts/install.sh` parses cleanly (no syntax errors).
3. `shellcheck scripts/install.sh` produces no warnings (or documented exclusions only).
4. Running on Ubuntu 24.04 with prerequisites met: each phase logs an info line and exits 0.
5. Re-running the script after success is a no-op (idempotent): user exists, dirs exist, env file exists, no errors.
6. Running with `EUID != 0` exits 1 with "must run as root".
7. Running on a non-Ubuntu host exits 1 with the right message.
8. `bash scripts/ci.sh` exits 0.
9. No file outside the explicit lists is modified.
