## Problem

Pass 1 (PR-054) was internal review. Pass 2 is active: attempt the attacks an actual adversary would try. Document method and outcome; file remediation MICRO PRs for any real findings.

## Scope

In scope:
- `docs/SECURITY-AUDIT-2026-Q2-pen.md` documenting:
  - Setup: deploy a clean install in a test container; run audits against it via curl from a separate host on the same LAN.
  - Test cases: malformed inputs (huge bodies, control characters), SQL injection attempts, command injection via path params, auth bypass (missing header, malformed Basic, base64 of empty creds), CSRF without header, rate-limit bypass (via X-Forwarded-For spoofing), audit-log tampering (can the API write fake operator-action events).
  - Each test case: command, expected response, actual response, finding (if any).
- Findings filed as MICRO PRs.

Out of scope:
- Network-layer attacks (ARP poisoning, etc.). Out of project scope.
- Physical attacks. Out of project scope.

## Implementation notes

Run the tests against a known-bad version too, if possible: roll back to a tagged commit and confirm the test catches the prior vulnerability that was fixed. This validates the test, not just the current code.

The 12-15 cases should cover:
1. Auth: empty Basic, wrong creds, malformed header, missing header.
2. CSRF: POST without token, with mismatched token, with whitelisted-path bypass attempt.
3. Rate limit: 100 quick auth failures, X-Forwarded-For spoofing.
4. Storcli injection: path params with shell metachars (FastAPI int parsing should reject; verify).
5. Replace-flow: POST replace/insert without prior offline; serial mismatch.
6. Maintenance: start with absurd duration values.
7. SQL injection: events filter category with SQL-ish input.
8. XSS: event message containing `<script>` tags; verify escape in templates.
9. Path traversal: GET /static/../etc/passwd.
10. Audit forgery: operator-action event written from outside the app (only possible via direct DB access; document threat model).

Quarterly re-run per §6: schedule a calendar reminder.
