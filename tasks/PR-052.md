## Problem

The Overview dashboard shows current state plus a 24-hour temperature chart. Operators investigating a thermal incident need longer windows: 7-day for weekly heat patterns, 30-day for seasonal drift, plus a per-drive panel grid for outlier identification.

## Scope

In scope:
- `grafana/dashboards/temperature-trends.json` with:
  - Time-range variable (24h / 7d / 30d quick-select).
  - Time-series chart: all drive temperatures, single panel, color per drive.
  - Time-series chart: RoC temperature with warning (95) and critical (105) horizontal annotation lines.
  - Time-series chart: max-temperature-across-drives (single line, easy to read).
  - Repeating panel: per-drive panel grid (one panel per drive), each panel showing min/avg/max temperature for the time range.
- Refresh every 30 s.
- Around 250 lines JSON.

Out of scope:
- Forecasting or anomaly detection. Operators eyeball the chart.
- Capacity planning view (drive-wear trends). PR-053 includes capacity dashboards.

## Files to create

`grafana/dashboards/temperature-trends.json`:
- UID `megaraid-temperature-trends`.
- Use Grafana's `repeat` feature on a stat panel keyed by `enclosure` and `slot` labels so adding a new drive adds a new panel automatically.
- Annotations for warning/critical thresholds: declared in the dashboard JSON's `annotations` section, queryable from the same Prometheus datasource.

```json
{
  "title": "Megaraid Temperature Trends",
  "uid": "megaraid-temperature-trends",
  "refresh": "30s",
  "templating": {
    "list": [
      {
        "name": "drive",
        "type": "query",
        "datasource": "Prometheus",
        "query": "label_values(megaraid_drive_temperature_celsius, slot)",
        "includeAll": true,
        "multi": true
      }
    ]
  },
  "panels": [
    /* All-drives temperature time-series */
    /* RoC temp time-series with thresholds */
    /* Max-temp time-series */
    /* Repeating per-drive panel */
  ]
}
```

## Implementation notes

The "repeating panel" feature regenerates panels at view time from the variable values. With the `slot` template variable resolved against the live Prometheus, every drive currently reporting metrics gets a panel. Drives that stop reporting (replaced, removed) drop out automatically.

Annotation lines for thresholds (95 / 105 RoC, 55 / 60 drive) are declared per panel in the `thresholds` config plus an annotation in `annotations.list`. Document the threshold values inline so the operator sees them on the chart.

The 30-day window stresses Prometheus less than expected because PR-048 retention is 90 days at 15 s scrape interval; the TSDB compacts at one-hour buckets after 24 hours, so 30-day queries are cheap.

UID convention: `megaraid-<topic>` so all dashboards group naturally in the Browse view.
