## Implementation notes

The Â§3 design system commits to "three font sizes maximum per page". The `:root` block defines exactly three: label / body / value. Anywhere a fourth size is tempting, refactor instead.

Token naming follows the existing `--name-modifier` convention. New tokens use the same prefix pattern (`--font-size-*`, `--space-*`).

The footer is small but deliberate. Operators reading the dashboard need to know which version they are looking at when a question arises. Without it, "is this the build that has the fix" is unanswerable. PR-059 will plumb real version + SHA; for now placeholders are fine.

The `--color-info` and `--color-neutral` tokens are added for Â§3.5 ("Five colors: optimal, warning, critical, info, neutral"). The existing CSS already has optimal/warning/critical; this PR completes the palette.

The "no hero typography" rule is enforceable in code review only. The unit test asserts that legacy class names are not in HTML output, which is a proxy. If the design starts drifting, raise it in Â§3 review at the next quarterly roadmap pass.

The hover transition `--transition-fast: 120ms ease-out` is the budget for the Â§3.4 "click response under 100 ms" rule. 120 ms covers paint plus transition without exceeding the perception threshold.
