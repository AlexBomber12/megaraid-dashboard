## Problem

PR-043 ships the exporter scaffold with a single up-gauge. This PR adds the per-drive metrics: temperature, physical-drive state (numeric encoding), virtual-drive state. These are the workhorse metrics for the Grafana drive dashboards (PR-052).

## Scope

In scope:
- New gauges in `web/metrics.py`, registered into the same registry:
  - `megaraid_drive_temperature_celsius{enclosure, slot, model, serial}`
  - `megaraid_physical_drive_state{enclosure, slot, model, serial}` (numeric encoding 0=optimal, 1=warning/rebuilding, 2=critical/failed/offline)
  - `megaraid_virtual_drive_state{vd_id, name, raid_level}` (same encoding)
- Each scrape reads the LATEST snapshot from the DB and updates the gauges.
- A small in-memory cache to avoid querying on every scrape if the snapshot has not changed (Prometheus may scrape every 15 s but the collector runs every 300 s; many scrapes return identical data).
- Tests: gauge values reflect a seeded snapshot; absent snapshot yields no metric values (NOT zero).

Out of scope:
- Controller-level metrics (RoC temp, CV capacitance). PR-045.
- Event / alert metrics. PR-046.
- High-cardinality labels (e.g. firmware version). Operators do not chart by firmware; keep label set tight.
- Histograms of temperature distribution. Counter / gauge are sufficient.

```python
from prometheus_client.core import GaugeMetricFamily

class MegaraidCollector:
    def __init__(self, session_factory: sessionmaker) -> None:
        self._session_factory = session_factory
        self._cache: tuple[int | None, list[GaugeMetricFamily]] = (None, [])

    def collect(self) -> Iterable[GaugeMetricFamily]:
        with self._session_factory() as session:
            latest_id = session.execute(
                select(ControllerSnapshot.id)
                .order_by(ControllerSnapshot.captured_at.desc())
                .limit(1)
            ).scalar_one_or_none()
        if latest_id is not None and latest_id == self._cache[0]:
            yield from self._cache[1]
            return
        families = self._build_families(latest_id)
        self._cache = (latest_id, families)
        yield from families

    def _build_families(self, latest_id: int | None) -> list[GaugeMetricFamily]:
        if latest_id is None:
            return []
        with self._session_factory() as session:
            snap = session.get(ControllerSnapshot, latest_id)
            if snap is None:
                return []
            pds = list(snap.physical_drives)
            vds = list(snap.virtual_drives)

        temp = GaugeMetricFamily(
            "megaraid_drive_temperature_celsius",
            "Per-physical-drive temperature in Celsius from latest snapshot.",
            labels=["enclosure", "slot", "model", "serial"],
        )
        pd_state = GaugeMetricFamily(
            "megaraid_physical_drive_state",
            "Per-physical-drive state (0=optimal, 1=warning, 2=critical).",
            labels=["enclosure", "slot", "model", "serial"],
        )
        vd_state = GaugeMetricFamily(
            "megaraid_virtual_drive_state",
            "Per-virtual-drive state (0=optimal, 1=warning, 2=critical).",
            labels=["vd_id", "name", "raid_level"],
        )

        for pd in pds:
            labels = [str(pd.enclosure_id), str(pd.slot_id), pd.model, pd.serial_number]
            if pd.temperature_celsius is not None:
                temp.add_metric(labels, float(pd.temperature_celsius))
            pd_state.add_metric(labels, float(_encode_pd_state(pd.state)))

        for vd in vds:
            labels = [str(vd.vd_id), vd.name, vd.raid_level]
            vd_state.add_metric(labels, float(_encode_vd_state(vd.state)))

        return [temp, pd_state, vd_state]


def _encode_pd_state(state: str) -> int:
    s = state.lower()
    if s in {"onln", "ugood", "optl"}:
        return 0
    if s in {"rbld", "ureb", "missing"}:
        return 1
    return 2  # offln, failed, ubad, etc.


def _encode_vd_state(state: str) -> int:
    s = state.lower()
    if s in {"optl", "optimal"}:
        return 0
    if s in {"dgrd", "degraded", "pdgd", "partially-degraded"}:
        return 1
    return 2  # offln, failed
```

## Implementation notes

State-encoding is opinionated. The mapping above is best-effort against storcli vocabulary; if a state appears that does not match any of the documented buckets, the encoder defaults to 2 (critical). This is fail-loud: a never-before-seen state should raise alarms.

Why labels include serial: a drive replaced in slot 2:0 has a different serial. Without serial, time-series for the new drive merges with the old one in Prometheus, and trends are misleading. Cardinality cost: one new time-series per replacement, which is fine.

The cache key is `latest_snapshot.id`. Same id = same data; we can return the previously-built families. Different id = new snapshot since the last scrape; rebuild.

Why a custom `Collector` instead of bare `Gauge`s: gauges are stateful and require explicit reset between scrapes. A `Collector` returning fresh families on every `collect()` is the idiomatic pattern for "metrics derived from a database read".

The label set `(enclosure, slot, model, serial)` for physical drives stays under 100 distinct combinations even with replacement churn over years. Prometheus is comfortable with this cardinality.

For VDs: `vd_id` is stable across snapshots; `name` is operator-set. Including both is fine; if `name` ever changes, the time-series resets, which is the right behavior.
