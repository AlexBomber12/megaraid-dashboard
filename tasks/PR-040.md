# PR-040: Maintenance mode storage

- Type: feature

- Depends on: PR-005

## Problem

When the operator performs maintenance on the array (drive replace, firmware flash, controller swap), the dashboard's notifier keeps sending alert emails because the array transiently looks degraded. The roadmap §7 open question 4 commits: "maintenance mode pauses notifier only; collector continues recording". This requires persistent state somewhere the collector and notifier can both see.

This PR ships only the storage layer: a `system_state` key-value table, an Alembic migration, and a small DAO. The endpoints and UI banner land in PR-041 and PR-042.

## Scope

In scope:
- New SQLAlchemy model `SystemState` with columns: `key: str` (primary key), `value: str`, `updated_at: datetime`.
- Alembic migration `0006_system_state` with both `upgrade()` and `downgrade()`.
- DAO functions: `get_state(session, key) -> str | None`, `set_state(session, key, value) -> None`, `delete_state(session, key) -> None`.
- A typed wrapper around the maintenance-mode key: `is_maintenance_mode_active(session) -> bool`, `set_maintenance_mode(session, *, active, expires_at)`, etc. The wrapper serializes a small JSON payload (`{"active": true, "expires_at": "...", "started_by": "..."}`) into the `value` column.
- Tests: migration round-trip, DAO get/set/delete, JSON payload round-trip on the typed wrapper.

Out of scope:
- Endpoints. PR-041.
- UI banner. PR-042.
- Notifier integration. PR-041.
- General-purpose feature flags via the same table. The schema is general; usage is scoped to maintenance mode for now.

`src/megaraid_dashboard/db/dao.py`:
- Add the three generic DAO functions plus the typed maintenance-mode wrapper.

```python
import json
from datetime import datetime, UTC

_MAINT_KEY = "maintenance_mode"


def get_state(session: Session, key: str) -> str | None:
    row = session.get(SystemState, key)
    return row.value if row else None


def set_state(session: Session, key: str, value: str) -> None:
    now = datetime.now(UTC)
    row = session.get(SystemState, key)
    if row is None:
        session.add(SystemState(key=key, value=value, created_at=now, updated_at=now))
    else:
        row.value = value
        row.updated_at = now


def delete_state(session: Session, key: str) -> None:
    row = session.get(SystemState, key)
    if row:
        session.delete(row)


@dataclass(frozen=True)
class MaintenanceState:
    active: bool
    expires_at: datetime | None
    started_by: str | None


def get_maintenance_state(session: Session, *, now: datetime) -> MaintenanceState:
    if now.tzinfo is None:
        raise ValueError("now must be timezone-aware")
    raw = get_state(session, _MAINT_KEY)
    if not raw:
        return MaintenanceState(active=False, expires_at=None, started_by=None)
    payload = json.loads(raw)
    expires_str = payload.get("expires_at")
    expires_at = datetime.fromisoformat(expires_str) if expires_str else None
    if expires_at and expires_at <= now:
        return MaintenanceState(active=False, expires_at=expires_at,
                                started_by=payload.get("started_by"))
    return MaintenanceState(
        active=bool(payload.get("active")),
        expires_at=expires_at,
        started_by=payload.get("started_by"),
    )


def set_maintenance_state(
    session: Session, *, active: bool, expires_at: datetime | None,
    started_by: str | None,
) -> None:
    if expires_at is not None and expires_at.tzinfo is None:
        raise ValueError("expires_at must be timezone-aware")
    if active:
        payload = {
            "active": True,
            "expires_at": expires_at.isoformat() if expires_at else None,
            "started_by": started_by,
        }
        set_state(session, _MAINT_KEY, json.dumps(payload))
    else:
        delete_state(session, _MAINT_KEY)
```

## Implementation notes

The key-value pattern is intentionally generic. If a future feature needs persistent global state (e.g. "last manual reload timestamp"), it can reuse the table without another migration. The maintenance-mode-specific wrapper sits on top.

JSON payload for `value` instead of separate columns: keeps the schema simple and forward-compatible. Adding fields to the maintenance state payload (e.g. a reason string) does not require a migration.

Auto-expiration is read-side: `get_maintenance_state` checks `expires_at <= now` and returns `active=False` even if the row says active. This avoids needing a janitor task to clean up expired rows. The next `set_maintenance_state(active=False, ...)` call deletes the row.

The DAO functions are pure SQLAlchemy; no business logic. The typed wrapper functions add the JSON serialization and expiry handling.

`expires_at` is required to be timezone-aware. The codebase already uses UTC consistently; preventing naive datetimes here matches that convention.

Risk heading in PR description: this PR adds a DB schema change per AGENTS.md security model rule.
