- Type: ux

## Problem

PR-041 ships the maintenance endpoints. Operators need a visible indicator on every page that maintenance mode is active, with a one-click stop button. A persistent banner at the top of every page (above the header strip from PR-018) is the clearest disposition.

## Scope

In scope:
- A banner partial `templates/partials/maintenance_banner.html` rendered conditionally in `templates/layouts/base.html` based on a view-model field `maintenance_state`.
- The banner shows: warning icon, "Maintenance mode active", remaining time countdown via JS, started-by username, Stop button.
- The Stop button posts to `/maintenance/stop` via htmx with CSRF.
- A "Start maintenance" affordance: probably NOT a banner button; instead a small section on a settings or admin sub-page. For this PR, the start affordance lives at the bottom of the overview page as a small form: duration dropdown (15/30/60/240/1440 min) + reason text + Start button.
- All page renders (overview, drives, drive-detail, events) include the banner.
- Tests: banner shows when active, hidden when inactive, stop button works, start form posts correctly.

Out of scope:
- A dedicated /admin or /settings page. The start form lives on overview for now.
- Editing an active maintenance window (extend duration, change reason). Operator stops and restarts.
- Push refresh of the banner without page reload. The 30 s overview auto-refresh covers it; the countdown JS updates locally.

`src/megaraid_dashboard/static/js/maintenance.js`:
- Around 30 lines. Updates the countdown text once per second.

```js
(function () {
  function fmt(seconds) {
    if (seconds <= 0) return "expired";
    const h = Math.floor(seconds / 3600);
    const m = Math.floor((seconds % 3600) / 60);
    const s = seconds % 60;
    if (h > 0) return h + "h " + m + "m left";
    if (m > 0) return m + "m " + s + "s left";
    return s + "s left";
  }

  function tickAll() {
    document.querySelectorAll("[data-maintenance-countdown]").forEach(function (el) {
      const expiresAt = new Date(el.getAttribute("data-expires"));
      const remaining = Math.floor((expiresAt - new Date()) / 1000);
      el.textContent = fmt(remaining);
    });
  }

  document.addEventListener("DOMContentLoaded", function () {
    tickAll();
    setInterval(tickAll, 1000);
  });
})();
```

## Implementation notes

The banner is `position: sticky; top: 0` so it remains visible as the operator scrolls. It is also `z-index: 100` to sit above modals or popovers if any exist later.

The countdown JS updates every second; this is cheap (one DOM mutation, one Date arithmetic) and gives the operator a clear sense of "how long left". If the page sits idle for the full duration, the countdown reaches "expired" and the next page reload (auto-refresh on overview) re-renders the banner as gone.

`hx-ext="json-enc"` makes htmx send the form data as JSON instead of form-urlencoded. This matches the FastAPI route's `MaintenanceStartRequest` body model.

The `window.location.reload()` after a successful start/stop is the simplest way to refresh banner state across all on-page elements (header, alert-status panel, timeline). htmx-driven partial swap of the banner alone would leave stale state elsewhere.

Why "started by" username appears in the banner: provides accountability and helps coordination if the homelab gains a second user. For a single-operator deploy it is informational only.
