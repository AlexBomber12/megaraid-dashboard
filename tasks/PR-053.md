## Note on target repository

This PR ships in `homelab-monitoring`, not megaraid-dashboard.

## Problem

PR-051 covers current state and short-window trends. PR-052 covers temperature trends. Two more analytics views remain: events-over-time (severity distribution, top categories) and capacity / wear (drive aging, free space, predicted full date if applicable).

## Scope

In scope:
- `grafana/dashboards/events.json`:
  - Stat: events in last 24h, broken down by severity.
  - Bar chart: events per category over the selected time range.
  - Time series: alerts-sent rate (per hour), useful to spot quiet/noisy windows.
  - Stat: collector last-run age (now minus `megaraid_collector_last_run_timestamp`); red if older than 600 s.
  - Stat: collector cycle duration (last value).
- `grafana/dashboards/capacity.json`:
  - Stat: total raid capacity vs used (if exposed; if not, document a TODO for the megaraid-dashboard exporter).
  - Time series: drive temperatures vs storcli's reported "media error count" if exposed; if not, just temperatures.
  - Per-drive table: model, serial, age (computed from first-seen-timestamp via Prometheus's `min_over_time`), current state.
  - Annotation: drive replacement events drawn as vertical lines on the time-series.
- Each dashboard around 250-300 lines JSON.
- Cross-link from Overview dashboard to Events dashboard via panel-row links.

Out of scope:
- SMART data integration. Megaraid-dashboard exposes some SMART alerts via events but not raw SMART attributes; exposing those is a follow-up to the exporter (not this repo).
- Predictive failure modeling. Beyond scope; eyeball the trends.

## Files to create

`grafana/dashboards/events.json`:
- UID `megaraid-events`.
- Panels reference `megaraid_events_total`, `megaraid_alerts_sent_total`, `megaraid_collector_*`.
- Use `rate()` and `increase()` PromQL for time-window aggregations.

`grafana/dashboards/capacity.json`:
- UID `megaraid-capacity`.
- Panels reference drive temperature series and any capacity gauges. Document gaps where exporter does not yet expose data.
- The "drive age" panel: PromQL `time() - min_over_time(megaraid_drive_temperature_celsius{slot=~".+"}[90d])` approximates "first time this drive reported in the 90-day window", which is a proxy for age. Document the approximation.

## Implementation notes

Capacity metrics are partially derived. The exporter (PR-044/045) currently emits state and temperature, not size or wear. Where this dashboard wants size, mark with a TODO panel and let it render "no data". The TODO drives a follow-up exporter PR.

The "rate of alerts" chart uses `rate(megaraid_alerts_sent_total[1h])`. With one alert per emitting incident, this is mostly zero; spikes correspond to operator-relevant events.

Drive replacement annotations: query `megaraid_events_total{category="operator_action"}` and overlay as vertical bars. Operators can correlate "after the drive was replaced, did the temperature pattern change".

Cross-links from Overview: Grafana supports per-panel "links" pointing at other dashboards by UID. Add three at the top of Overview pointing at the three other dashboards.
