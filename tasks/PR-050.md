## Note on target repository

This PR ships in `homelab-monitoring`, not megaraid-dashboard.

## Problem

PR-047/048/049 bind Prometheus and Grafana to loopback. Operator can SSH-tunnel for now, but the long-term disposition is HTTPS access via the same `*.alexbomber.com` domain. Caddy is the chosen reverse proxy for this stack: automatic HTTPS via Let's Encrypt, simple Caddyfile, basic-auth plugin built-in.

## Scope

In scope:
- A `caddy` service in `docker-compose.yml`.
- `caddy/Caddyfile` with TLS for `monitoring.alexbomber.com`, basic auth (single user), proxy to grafana:3000 on `/`, proxy to prometheus:9090 on `/prometheus/*`.
- Auto-HTTPS: Caddy fetches Let's Encrypt cert on first start.
- Persistent cert volume.
- README updated with DNS and firewall prerequisites.

Out of scope:
- Multi-user auth. Single operator.
- Forward-auth / OAuth. Basic auth is enough.
- Path-based routing for additional services. Add when needed.

## Files to create

`caddy/Caddyfile`:
```
{
    email aleksei@example.com
    admin off
}

monitoring.alexbomber.com {
    encode zstd gzip

    header {
        Strict-Transport-Security "max-age=63072000; includeSubDomains"
        X-Content-Type-Options "nosniff"
        Referrer-Policy "strict-origin-when-cross-origin"
        X-Frame-Options "DENY"
    }

    basicauth /* {
        {$BASIC_AUTH_USER} {$BASIC_AUTH_PASSWORD_HASH}
    }

    handle_path /prometheus/* {
        reverse_proxy prometheus:9090
    }

    handle {
        reverse_proxy grafana:3000
    }
}
```

## Files to modify

`docker-compose.yml`:
- Add caddy service:

```yaml
caddy:
  image: caddy:2.8-alpine
  container_name: homelab-caddy
  restart: unless-stopped
  ports:
    - "443:443"
    - "80:80"
  environment:
    BASIC_AUTH_USER: "${CADDY_BASIC_AUTH_USER:?required}"
    BASIC_AUTH_PASSWORD_HASH: "${CADDY_BASIC_AUTH_PASSWORD_HASH:?required}"
  volumes:
    - "/srv/homelab-monitoring/caddy/Caddyfile:/etc/caddy/Caddyfile:ro"
    - "/srv/homelab-monitoring/caddy/data:/data"
    - "/srv/homelab-monitoring/caddy/config:/config"
  depends_on:
    - grafana
    - prometheus
  networks:
    - monitoring
```

## Implementation notes

Caddy 2.8 is the current major (verified at roadmap version). Auto-HTTPS works out of the box for any domain whose DNS resolves to the host on port 80.

`{$BASIC_AUTH_PASSWORD_HASH}` is a bcrypt hash, NOT plaintext. Generate via `docker run --rm caddy:2.8-alpine caddy hash-password --plaintext 'your-password'` and paste into `.env`.

The `/prometheus/*` path strips the prefix via `handle_path` (note the trailing slash on rewriting): inside the container, Prometheus sees `/api/v1/...` paths normally.

`admin off` disables Caddy's own admin API surface (we never use it; closing reduces attack surface).

Risk heading in PR description: this PR opens external HTTPS surface for the first time in this repo. Mitigation: bcrypt-hashed password, HSTS, X-Frame-Options DENY, no admin API exposed.
