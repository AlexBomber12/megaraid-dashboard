- Complexity: low

## Problem

PR-044 ships per-drive metrics. This PR adds controller-level metrics: overall controller health, RoC temperature, and CV (cache vault / battery backup) capacitance percent. These power the Grafana overview dashboard's controller widgets (PR-051).

## Scope

In scope:
- Three new metrics in the existing collector:
  - `megaraid_controller_health{model, serial}` (gauge, 0/1/2 encoded)
  - `megaraid_controller_roc_temperature_celsius{model, serial}`
  - `megaraid_cv_capacitance_percent{model, serial}` (only emitted when CV present)
- Health derivation: optimal=0 when controller alarm is off and no critical drives; warning=1 when alarm off but any warning state; critical=2 when alarm on or any critical drives. Reuse the same aggregation logic from the dashboard tile (PR-020) by extracting it into a shared helper.
- Tests: each metric value matches the seeded snapshot's expectations.

Out of scope:
- BBU metrics (deprecated; CV is the modern replacement). If a deploy still has BBU, add later.
- Per-controller-card metrics for multi-controller deploys. Single-controller per §2.7.
- Firmware version metric. Operators do not chart by firmware; documented in §2.

```python
controller_health = GaugeMetricFamily(
    "megaraid_controller_health",
    "Controller health (0=optimal, 1=warning, 2=critical).",
    labels=["model", "serial"],
)
roc_temp = GaugeMetricFamily(
    "megaraid_controller_roc_temperature_celsius",
    "Controller RoC silicon temperature in Celsius.",
    labels=["model", "serial"],
)
cv_capacitance = GaugeMetricFamily(
    "megaraid_cv_capacitance_percent",
    "Cache vault capacitance percent.",
    labels=["model", "serial"],

ctrl_labels = [snap.model_name, snap.serial_number]
controller_health.add_metric(ctrl_labels, float(_encode_controller_health(snap, pds, vds)))
if snap.roc_temperature_celsius is not None:
    roc_temp.add_metric(ctrl_labels, float(snap.roc_temperature_celsius))
cv = snap.cachevault
if cv is not None and cv.capacitance_percent is not None:
    cv_capacitance.add_metric(ctrl_labels, float(cv.capacitance_percent))
```

## Implementation notes

Controller health derivation should be the SAME logic the dashboard tile uses, otherwise Grafana's panel and the dashboard's tile disagree. If the helper is in `services/overview.py`, import it from `web/metrics.py`. If not yet refactored to a shared helper, that is a small refactor follow-up; for THIS PR, copy the rule. Document the duplication with a comment so the next maintainer collapses it.

CV capacitance absence (snap.cachevault is None) is a hardware reality: not every controller has a CV module. Skip the metric line entirely rather than emitting NaN or 0; Prometheus interprets a missing series as "no data", which is correct.

The label set `(model, serial)` for the controller is invariant per host (single controller). Cardinality is 1.

Risk heading not strictly required (no schema, no auth, no storcli wrapper change). Document why in PR description.
