## Problem

The repo is MIT-licensed and public. To accept contributions or be a useful reference for someone else's homelab, it needs the standard open-source meta-files: CONTRIBUTING.md, CODE_OF_CONDUCT.md, issue and PR templates. None are required for a one-author project; they are the cost of admission for community engagement.

## Scope

In scope:
- `CONTRIBUTING.md`: how to set up dev env, run tests, submit a PR, expected style. References `AGENTS.md` for style and `INSTALL.md` for environment.
- `CODE_OF_CONDUCT.md`: Contributor Covenant 2.1 verbatim.
- `.github/ISSUE_TEMPLATE/bug_report.md` and `feature_request.md`.
- `.github/PULL_REQUEST_TEMPLATE.md`: short template referencing the task-file format and CI requirements.
- README badges row: license, CI status.

Out of scope:
- GitHub Discussions enablement.
- Sponsorship setup.
- Translation files.

## Files to create

`CONTRIBUTING.md` (around 150 lines):
- Sections: Reporting bugs, Suggesting features, Setting up dev env, Code style, Testing, Submitting a PR, Working with the orchestrator (link to AGENTS.md daemon-managed sections).

`CODE_OF_CONDUCT.md`:
- Contributor Covenant 2.1, with reporting email filled in.

`.github/ISSUE_TEMPLATE/bug_report.md`:
- Sections: Description, Steps to reproduce, Expected, Actual, Environment, Logs.

`.github/ISSUE_TEMPLATE/feature_request.md`:
- Sections: Problem, Proposed solution, Alternatives considered.

`.github/PULL_REQUEST_TEMPLATE.md`:
- Sections: PR_ID (cross-ref task file), What changed, How verified, Risk heading (when applicable per AGENTS.md).

## Files to modify

`README.md`:
- Badges row at top: license, CI status, latest release tag (when PR-059 establishes one).
- Link to CONTRIBUTING.md and CODE_OF_CONDUCT.md in a Community section.

## Success criteria

1. All five new files exist with documented sections.
2. README has badges and Community links.
3. GitHub renders the templates correctly when opening a new issue / PR (manual verification).
4. CONTRIBUTING references AGENTS.md and INSTALL.md.
5. CI passes.
6. No file outside the explicit lists is modified.
