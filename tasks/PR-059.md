# PR-059: Versioning + reproducible builds

Branch: pr-059-versioning-builds
- Type: config
- Complexity: low
- Depends on: none
- Priority: 5
- Coder: any

## Problem

The footer placeholder from PR-018 reads `Version dev | Build unknown`. Operators looking at the dashboard cannot tell which commit they are running. Reproducible builds (same commit → same artifact) are the precondition for any future "is this the version that has the fix" conversation.

## Scope

In scope:
- `__version__` in `src/megaraid_dashboard/__init__.py` populated from `importlib.metadata.version("megaraid-dashboard")` so the value tracks `pyproject.toml`.
- `pyproject.toml` `version = "0.1.0"`, with semver progression on each tagged release.
- Build SHA: read from `GIT_SHA` env var (set by deploy script via `git rev-parse HEAD`); fall back to "unknown".
- A small Makefile or CI step that builds a wheel artifact (`python -m build`) and asserts the produced wheel filename contains the right version.
- Footer template uses both values.

Out of scope:
- SBOM (software bill of materials). Add later if needed.
- Tag-based release automation in GitHub Actions. Manual `git tag` is sufficient.
- Multi-arch wheels. Pure-python package; one wheel works everywhere.

## Files to modify

`pyproject.toml`:
- Set `version = "0.1.0"`. Add `[project]` `dynamic = []` removed if previously dynamic.
- Add `build-system` with `requires = ["setuptools>=61", "wheel"]` if not already.

`src/megaraid_dashboard/__init__.py`:
```python
from importlib.metadata import PackageNotFoundError, version as _pkg_version

try:
    __version__ = _pkg_version("megaraid-dashboard")
except PackageNotFoundError:
    __version__ = "0.0.0+unknown"
```

`src/megaraid_dashboard/web/templates.py`:
- Wire `app_version` and `build_sha` as Jinja globals:

```python
import os
env.globals["app_version"] = __version__
env.globals["build_sha"] = (os.environ.get("GIT_SHA") or "unknown")[:8]
```

`scripts/install.sh`:
- Capture `GIT_SHA="$(cd "${repo_root}" && git rev-parse HEAD)"` if `.git` is present, write it into `${ENV_FILE}`.

`tests/test_versioning.py` (new):
- Test: `from megaraid_dashboard import __version__` returns a string matching `\d+\.\d+\.\d+(\+\w+)?`.
- Test: rendered footer contains the version.

## Files NOT to touch

- `services/`, `db/`. No business change.

## Success criteria

1. `python -c "from megaraid_dashboard import __version__; print(__version__)"` prints `0.1.0`.
2. Building a wheel via `python -m build` produces `megaraid_dashboard-0.1.0-*.whl`.
3. With `GIT_SHA=abc1234567` set, the rendered footer shows `Version 0.1.0 | Build abc12345` (8 chars).
4. With `GIT_SHA` unset, the footer shows `Version 0.1.0 | Build unknown`.
5. CI passes.
6. No file outside the explicit lists is modified.
