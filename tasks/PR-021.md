## Problem

PR-004 shipped the alert-status panel as a `<table>`-style block: four labeled rows, label-on-left, value-on-right, taking around 200 px of vertical space below the new six-tile strip. Under §3 information-density rules this is too tall for a panel showing four numbers and one badge. The redesign collapses it into a single horizontal row of four cells fitting in around 60 px vertical.

## Scope

In scope:
- Refactor `templates/partials/alert_status.html` to render a 4-cell horizontal grid instead of a table/dl layout.
- The four cells: Last sent (relative time + tooltip with absolute), Pending (count + warning color when > 0), Sent in last hour (count), Notifier health (badge + label).
- Reuse `.status-tile` for visual consistency where appropriate. If the tile shape is too large, define a smaller `.alert-cell` variant inheriting tokens from PR-018.
- Match the hover and color contract from §3.5.
- Tests updated to assert the new structure.

Out of scope:
- Changing `_load_alert_status` business logic. The view-model from PR-004 stays exactly the same.
- Real-time updates beyond the existing 30 s auto-refresh.
- Per-recipient breakdown.

## Implementation notes

Grid with `repeat(4, minmax(160px, 1fr))` gives consistent column widths and graceful narrow-viewport reflow (the cells wrap or shrink). The fixed minmax floor matches `--tile-min-width` from PR-018, so visually the alert cells align with the status strip above.

The "Last sent" cell uses `data-local-time` from PR-010, so the displayed timestamp localizes via JS. The fallback inner text "Never" is rendered when the value is None; otherwise the fallback is the UTC-suffixed ISO string.

The pending-count cell tints warning when count > 0, neutral when zero. This is information density: the operator does not need to read the number to know there are pending events; the color signals it.

The notifier-health cell uses both the colored left border AND the inline status badge. Two indicators, one screen-area, no ambiguity.

The vertical space saved versus PR-004's table layout is around 140 px on 1080p. That space is available below for the activity timeline (PR-022) and ensures both fit above the fold.
