# PR-061: Pre-commit hooks

- Depends on: none

## Problem

Developers and AI coders sometimes push commits that fail CI on trivial issues (ruff format, mypy, missing newline). pre-commit catches these locally before the push and tightens the feedback loop.

## Scope

In scope:
- `.pre-commit-config.yaml` with: ruff (check + format), mypy (subset, fast mode), shellcheck on `scripts/*.sh`, yamllint, end-of-file-fixer, trailing-whitespace.
- README and CONTRIBUTING.md updated with `pre-commit install` instruction.
- pre-commit is OPTIONAL: CI still validates everything, the hook is just developer-side QoL.

Out of scope:
- Mandatory enforcement (would require server-side hook). Optional is enough.
- pyright, black, autopep8, etc. ruff is the only linter / formatter per `AGENTS.md`.

## Files to create

`.pre-commit-config.yaml`:
```yaml
repos:
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v5.0.0
    hooks:
      - id: end-of-file-fixer
      - id: trailing-whitespace
      - id: check-yaml
      - id: check-json
      - id: check-added-large-files
        args: ["--maxkb=500"]

  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.7.4
    hooks:
      - id: ruff
        args: [--fix]
      - id: ruff-format

  - repo: https://github.com/pre-commit/mirrors-mypy
    rev: v1.13.0
    hooks:
      - id: mypy
        files: ^src/
        additional_dependencies:
          - pydantic>=2
          - sqlalchemy>=2

  - repo: https://github.com/shellcheck-py/shellcheck-py
    rev: v0.10.0.1
    hooks:
      - id: shellcheck
        files: \.sh$

  - repo: https://github.com/adrienverge/yamllint
    rev: v1.35.1
    hooks:
      - id: yamllint
```

## Files to modify

`README.md`:
- One paragraph: "Pre-commit (optional): `pip install pre-commit && pre-commit install`. Hooks run on `git commit`."

`CONTRIBUTING.md` (from PR-060 if merged; otherwise note as TODO):
- Add a "Pre-commit setup" subsection.

## Success criteria

1. `.pre-commit-config.yaml` validates: `pre-commit validate-config`.
2. `pre-commit run --all-files` passes on a clean repo.
3. README documents the install command.
4. CI does NOT depend on pre-commit (the hooks duplicate CI checks; CI is the canonical gate).
5. No file outside the explicit lists is modified.
