# PR-062: HTML email alternative

Branch: pr-062-html-email-alternative
- Type: feature
- Complexity: low
- Depends on: PR-053
- Priority: 5
- Coder: any

## Problem

Alert emails are plain text. After Grafana dashboards land (PR-051..053), an email could embed a small HTML chart or dashboard link that previews state at the moment of the alert. The roadmap notes this PR may become redundant after Grafana dashboards exist; ship it only if HTML emails add value beyond the dashboard link.

## Scope

In scope (only if the operator confirms desired):
- HTML alternative part for alert emails: subject unchanged, multipart/alternative MIME with a small HTML body that includes severity badge, message, link to Grafana Overview dashboard, link to event detail on the dashboard.
- Plain-text part remains unchanged.
- A config flag `alert_email_html_enabled: bool = false` (default off; opt-in).

Out of scope:
- Embedded chart images. CSP and email-client compatibility nightmare; Grafana link is sufficient.
- Theming or branding.

## Files to modify

`src/megaraid_dashboard/alerts/transport.py` (or wherever the email is composed):
- Compose the multipart message: text/plain part remains; new text/html part attached when `settings.alert_email_html_enabled`.
- Small Jinja template `templates/email/alert.html` (around 50 lines) for the HTML body.

`src/megaraid_dashboard/config.py`:
- Add `alert_email_html_enabled: bool = False`.

`src/megaraid_dashboard/templates/email/alert.html` (new):
- Minimal inline-styled HTML; no external CSS (email clients strip it).

`tests/test_alerts/test_transport_html.py`:
- With flag off, the sent email is text/plain only.
- With flag on, the email is multipart/alternative with both parts.
- HTML body includes the dashboard link and the event message.

## Implementation notes

If at PR-062 time the Grafana dashboards (PR-051..053) provide enough operator value, this PR can be marked WONTDO and the resource saved. Decision is the operator's; document the decision in the PR description.

The HTML body is small and inline-styled because email-client CSS support is famously bad. Outlook, Gmail, Proton Mail all strip `<style>` blocks; inline `style=""` is the lowest common denominator.

The dashboard link is a deep link to the Overview dashboard with a time range matching the event's occurrence, e.g. `https://monitoring.alexbomber.com/d/megaraid-overview?from=<event_time-1h>&to=<event_time+10min>`. Construct from `Settings.grafana_url` (new optional field) and the event's occurred_at.

## Success criteria

1. Flag default off; existing emails unchanged.
2. With flag on, multipart email contains both text and HTML parts.
3. HTML body includes severity, message, and Grafana dashboard link.
4. Tests cover both modes.
5. CI passes.
6. No file outside the explicit lists is modified.
