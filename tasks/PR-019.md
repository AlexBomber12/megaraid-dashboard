## Problem

PR-018 added the design tokens for tile dimensions (`--tile-padding`, `--tile-radius`, `--tile-min-width`, `--tile-height`). The actual tile component is the building block for PR-020 (six-tile status strip), PR-021 (compact alert status row), and any future at-a-glance UI. It needs to ship as a reusable Jinja partial plus the matching CSS rules, with a one-place definition that all callers use.

The component vocabulary in roadmap §3.3 calls it `StatusTile`: fixed dimensions, label uppercase top, value below, optional icon at left, colored left-border for state. The Lucide icon subset (§3.4) ships in this PR as an inline SVG sprite (max 4 KB total), referenced via `<svg><use href="#icon-name"/></svg>`.

## Scope

In scope:
- A new partial `templates/partials/status_tile.html` accepting parameters: `label`, `value`, `status` (one of optimal/warning/critical/info/neutral), `icon` (optional, one of the Lucide subset names), `href` (optional, makes the tile a link).
- New SVG sprite `static/icons.svg` containing the Lucide icons listed in §3.4: check-circle, alert-triangle, x-circle, help-circle, info, bell, bell-off, hard-drive, cpu, thermometer, lightbulb, refresh-cw, clock. Inlined as a `<svg><symbol id="icon-check-circle" viewBox="..."><path d="..."/></symbol>...</svg>` document; total under 4 KB.
- CSS rules in `app.css` for `.status-tile`, `.status-tile__label`, `.status-tile__value`, `.status-tile__icon`, `.status-tile--{state}` modifiers using the `var(--tile-*)` tokens from PR-018.
- Hover behavior: brightness shift only, no hue (per §3.5).
- A small reference page or test that renders one of every state, used in the test below to assert visual contract.

Out of scope:
- Status strip composition. PR-020.
- Replacing existing tiles in the overview page. PR-020.
- Tooltip with thresholds. PR-027.
- Mobile responsive variant. PR-063 (P2).

```jinja

{% set tile_tag = 'a' if href else 'div' %}

  class="status-tile status-tile--{{ tile_status }}"
  {% if href %}href="{{ href }}"{% endif %}
```

## Implementation notes

The `<use href>` pattern with an external SVG file means the icons are cached separately and reused across the page without inflating each tile render. The first request to a page fetches `icons.svg` once; all subsequent tile renders reference symbols by id without HTTP cost.

The 4 KB cap on the sprite is a soft commitment from §3.4. Lucide icons are around 200 bytes each in stripped form; 13 × 200 = 2600 bytes plus SVG wrapper. Achievable.

The partial accepts an optional `aria_label` for cases where `label + value` does not communicate well to a screen reader (e.g. "Controller / Optimal" reads ambiguously). If unset, screen readers fall back to reading the visible text, which is the right default.

The left-border state pattern (4 px wide) mirrors the §3.3 spec. Combined with the icon and the value, the operator gets three independent indicators of state (color, icon shape, text) per §3.5 "color is never the sole carrier of meaning".

Hover brightness only, never hue. The `--surface-hover: rgba(255, 255, 255, 0.04)` token from PR-018 lifts the background by approximately 5 percent on hover, which is enough to acknowledge interactivity without competing with the state color.

Lucide icon import: pick the smallest variant (no fill, no rounded caps unless mandated) so the sprite stays under 4 KB. The path data comes from `lucide-static` npm package OR direct copy from the GitHub repo. Cite the source in a header comment for license attribution (MIT requires attribution).
