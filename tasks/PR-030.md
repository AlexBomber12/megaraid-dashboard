# PR-030: install.sh: config wizard (interactive)

Branch: pr-030-install-config-wizard
- Type: feature
- Complexity: medium
- Depends on: PR-029
- Priority: 4
- Coder: any

## Problem

PR-028/PR-029 set up the system. Stage three populates `${ETC_DIR}/env` with the operator's specific configuration: admin password (bcrypt-hashed), SMTP host/user/token, alert recipient. Without this stage, the installer leaves an empty env file and the operator has to remember the schema and edit it by hand.

This phase is interactive by default with a `--non-interactive` flag for CI scenarios. Each prompt has a sane default where one exists; secrets are read with `read -s` to avoid echoing.

## Scope

In scope:
- New `phase_config` function in `scripts/install.sh`. Reads the existing `${ENV_FILE}`; for each missing key, prompts the operator.
- Inputs: `ADMIN_USERNAME`, `ADMIN_PASSWORD` (hashed via bcrypt before write), `ALERT_SMTP_HOST`, `ALERT_SMTP_PORT`, `ALERT_SMTP_USER`, `ALERT_SMTP_PASSWORD`, `ALERT_FROM`, `ALERT_TO`, `STORCLI_PATH`, `LOG_LEVEL`, `METRICS_INTERVAL_SECONDS`.
- Defaults: `ADMIN_USERNAME=admin`, `ALERT_SMTP_PORT=587`, `STORCLI_PATH=/usr/local/sbin/storcli64`, `LOG_LEVEL=info`, `METRICS_INTERVAL_SECONDS=300`.
- Bcrypt hashing via the installed venv (`${INSTALL_PREFIX}/.venv/bin/python -c "import bcrypt; ..."`).
- File mode preserved at 0600, owner root, group megaraid (per PR-028).
- Non-interactive mode reads values from environment variables (`MEGARAID_INSTALL_ADMIN_PASSWORD=...` etc.) and skips prompts. Missing required values fail with a clear list.
- Idempotency: existing values are preserved unless `--force-reconfigure` is passed.
- Tests: shellcheck plus a non-interactive test using env-var mode.

Out of scope:
- TUI / dialog. Plain `read -p` prompts.
- Validation of SMTP credentials (test send). The CLI test command from the email-alerts series can be invoked separately.
- Password strength enforcement beyond a minimum length.
- Encrypted env-file storage. mode 0600 plus filesystem permissions is the model.

## Files to modify

`scripts/install.sh`:
- Append:

```bash
NON_INTERACTIVE="false"
FORCE_RECONFIG="false"
for arg in "$@"; do
  case "${arg}" in
    --non-interactive) NON_INTERACTIVE="true" ;;
    --force-reconfigure) FORCE_RECONFIG="true" ;;
  esac
done

prompt() {
  # prompt VAR_NAME PROMPT [default] [secret]
  local var="$1" msg="$2" default="${3:-}" secret="${4:-}"
  local current
  current="$(grep -E "^${var}=" "${ENV_FILE}" 2>/dev/null | cut -d= -f2- || true)"
  if [[ -n "${current}" && "${FORCE_RECONFIG}" != "true" ]]; then
    log_info "${var} already set, keep"
    eval "${var}=$(printf '%q' "${current}")"
    return
  fi
  local env_var="MEGARAID_INSTALL_${var}"
  if [[ -n "${!env_var:-}" ]]; then
    eval "${var}=$(printf '%q' "${!env_var}")"
    return
  fi
  if [[ "${NON_INTERACTIVE}" == "true" ]]; then
    [[ -n "${default}" ]] || log_fail "${var} required in non-interactive mode (set ${env_var})"
    eval "${var}=$(printf '%q' "${default}")"
    return
  fi
  local input
  if [[ -n "${secret}" ]]; then
    read -r -s -p "${msg}: " input
    echo
  else
    read -r -p "${msg}${default:+ [${default}]}: " input
  fi
  [[ -n "${input}" ]] || input="${default}"
  [[ -n "${input}" ]] || log_fail "${var} required"
  eval "${var}=$(printf '%q' "${input}")"
}

bcrypt_hash() {
  local plain="$1"
  "${INSTALL_PREFIX}/.venv/bin/python" - <<PY
import bcrypt, sys
print(bcrypt.hashpw("${plain}".encode(), bcrypt.gensalt()).decode())
PY
}

phase_config() {
  log_info "Phase 7: config wizard"
  prompt ADMIN_USERNAME "admin username" "admin"
  prompt ADMIN_PASSWORD "admin password" "" "secret"
  prompt ALERT_SMTP_HOST "SMTP host"
  prompt ALERT_SMTP_PORT "SMTP port" "587"
  prompt ALERT_SMTP_USER "SMTP user"
  prompt ALERT_SMTP_PASSWORD "SMTP password / app token" "" "secret"
  prompt ALERT_FROM "from address"
  prompt ALERT_TO "to address"
  prompt STORCLI_PATH "storcli path" "/usr/local/sbin/storcli64"
  prompt LOG_LEVEL "log level" "info"
  prompt METRICS_INTERVAL_SECONDS "collector interval seconds" "300"

  local admin_hash
  admin_hash="$(bcrypt_hash "${ADMIN_PASSWORD}")"

  install -m 0600 -o root -g "${INSTALL_USER}" /dev/null "${ENV_FILE}.tmp"
  cat > "${ENV_FILE}.tmp" <<EOF
ADMIN_USERNAME=${ADMIN_USERNAME}
ADMIN_PASSWORD_HASH=${admin_hash}
ALERT_SMTP_HOST=${ALERT_SMTP_HOST}
ALERT_SMTP_PORT=${ALERT_SMTP_PORT}
ALERT_SMTP_USER=${ALERT_SMTP_USER}
ALERT_SMTP_PASSWORD=${ALERT_SMTP_PASSWORD}
ALERT_SMTP_USE_STARTTLS=true
ALERT_FROM=${ALERT_FROM}
ALERT_TO=${ALERT_TO}
STORCLI_PATH=${STORCLI_PATH}
STORCLI_USE_SUDO=true
LOG_LEVEL=${LOG_LEVEL}
METRICS_INTERVAL_SECONDS=${METRICS_INTERVAL_SECONDS}
DATABASE_URL=sqlite:///${DATA_DIR}/megaraid.db
EOF
  chmod 0600 "${ENV_FILE}.tmp"
  chown "root:${INSTALL_USER}" "${ENV_FILE}.tmp"
  mv "${ENV_FILE}.tmp" "${ENV_FILE}"
  log_info "wrote ${ENV_FILE}"
}
```

## Implementation notes

The `prompt` helper uses `eval` carefully via `printf %q` to safely interpolate user input into the variable assignment. Without `printf %q`, a password containing single quotes would break the eval.

The bcrypt hash uses the `bcrypt` Python package installed in the venv (PR-005 added `bcrypt>=4.1` to dependencies). Calling out to Python from bash avoids requiring bash-native bcrypt.

The temp-file pattern (`${ENV_FILE}.tmp` then `mv`) is atomic on POSIX filesystems: if the install is interrupted mid-write, the operator does not end up with a half-written env file.

Secrets are echoed only as `***` style after the read; the script does not log values back. Bash history is cleared by the leading space on `read` (interactive shells only); the script itself does not write secrets to logs.

Default for `STORCLI_USE_SUDO` is `true` because the production deploy runs the app as the `megaraid` user (no direct sudo right except via the narrow sudoers rule from PR-031).

`ALERT_SMTP_USE_STARTTLS=true` is hardcoded; if the user has Proton SMTP that uses different (port 465 SSL), expose it as a wizard prompt in a follow-up. For now Proton supports STARTTLS on 587.

The temp env file uses `set -e` semantics: any failure (cat redirect, mv) aborts. The `${ENV_FILE}` is never partially written.

7. The written env file passes `Settings()` validation when loaded by the application.
