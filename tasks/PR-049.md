## Note on target repository

This PR ships in `homelab-monitoring`, not megaraid-dashboard.

## Problem

Grafana from PR-047 starts up empty. Operator has to manually click through datasource setup. PR-049 adds file-based provisioning so Prometheus is wired automatically on first start, plus a `default` organization stub.

## Scope

In scope:
- `grafana/provisioning/datasources/prometheus.yml` declaring the Prometheus datasource at `http://prometheus:9090` (Compose network resolves the service name).
- `grafana/provisioning/dashboards/default.yml` declaring a file-based dashboard provider that loads JSON from `/var/lib/grafana/dashboards/`.
- `grafana/dashboards/.gitkeep` placeholder; actual JSON dashboards land in PR-051..053.
- README section: how to access Grafana on first boot, default credentials, where to add custom dashboards.

Out of scope:
- Notification channels (Alertmanager, email-to-Grafana). Skipped per §7 question 5.
- LDAP / OAuth. Single operator; basic auth on Caddy (PR-050) handles access.
- Dashboard JSON itself.

## Files to create

`grafana/provisioning/datasources/prometheus.yml`:
```yaml
apiVersion: 1

datasources:
  - name: Prometheus
    type: prometheus
    access: proxy
    url: http://prometheus:9090
    isDefault: true
    editable: false
    jsonData:
      timeInterval: 15s
      httpMethod: POST
```

`grafana/provisioning/dashboards/default.yml`:
```yaml
apiVersion: 1

providers:
  - name: default
    orgId: 1
    folder: ""
    type: file
    disableDeletion: false
    updateIntervalSeconds: 30
    allowUiUpdates: false
    options:
      path: /var/lib/grafana/dashboards
      foldersFromFilesStructure: true
```

`grafana/dashboards/.gitkeep`: empty file.

## Files to modify

`docker-compose.yml`:
- Update grafana service: bind-mount `grafana/dashboards/` to `/var/lib/grafana/dashboards/` so file-based provisioning sees it.


`README.md`:
- Add Grafana access section: "Visit `http://localhost:3000`, log in as `admin` with `${GRAFANA_ADMIN_PASSWORD}`. Prometheus datasource is provisioned automatically."

## Implementation notes

`editable: false` on the datasource prevents in-UI changes that would drift from the file. If the operator needs to experiment, edit the YAML file and restart.

`allowUiUpdates: false` on the dashboard provider is intentional: dashboards live in git. UI edits get overwritten on next reload, which is the correct disposition for "git is the source of truth".

`updateIntervalSeconds: 30` means a dashboard JSON change is reflected within 30 s of saving the file. No restart needed.
