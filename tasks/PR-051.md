## Note on target repository

This PR ships in `homelab-monitoring`, not megaraid-dashboard.

## Problem

The exporter in megaraid-dashboard surfaces metrics; Prometheus scrapes them; Grafana renders nothing yet. This PR adds the first dashboard: a single-screen Overview that mirrors the megaraid-dashboard's six-tile strip plus a few historical charts. Operator opens Grafana, lands on this dashboard, sees system state at a glance.

## Scope

In scope:
- `grafana/dashboards/overview.json` with these panels arranged on one screen:
  - Stat: Controller health (numeric, color-mapped).
  - Stat: RoC temperature (with threshold colors at 95 and 105).
  - Stat: max physical-drive temperature.
  - Stat: VD aggregate state (count of optimal vs degraded vs failed).
  - Stat: CV capacitance percent.
  - Time series: physical-drive temperatures (24-hour window).
  - Time series: RoC temperature (24-hour window).
  - Stat: alerts sent in the last 24 hours.
- Variable for time range (default 24h).
- Refresh every 30 s.
- The dashboard JSON is around 300 lines.

Out of scope:
- Drill-down dashboards. PR-052 (temperature trends), PR-053 (events).
- Alerting on dashboard panels. Skipped per §7 question 5.
- Per-drive panels with one chart per drive. Use the multi-line chart instead.

## Files to create

`grafana/dashboards/overview.json`:
- Use the Grafana JSON model (export-from-UI format). Skeleton:

```json
{
  "title": "Megaraid Overview",
  "uid": "megaraid-overview",
  "tags": ["megaraid", "homelab"],
  "timezone": "browser",
  "refresh": "30s",
  "time": { "from": "now-24h", "to": "now" },
  "panels": [
    {
      "id": 1, "type": "stat", "title": "Controller health",
      "gridPos": { "h": 4, "w": 4, "x": 0, "y": 0 },
      "targets": [{ "expr": "megaraid_controller_health", "refId": "A" }],
      "fieldConfig": {
        "defaults": {
          "thresholds": {
            "mode": "absolute",
            "steps": [
              { "color": "green", "value": null },
              { "color": "yellow", "value": 1 },
              { "color": "red",    "value": 2 }
            ]
          },
          "mappings": [
            { "type": "value", "options": { "0": { "text": "Optimal" }, "1": { "text": "Warning" }, "2": { "text": "Critical" } } }
          ]
        }
      }
    }
    /* additional panels follow the same shape */
  ],
  "schemaVersion": 39,
  "version": 1
}
```

- Each Stat panel uses thresholds + value mappings to render text labels (Optimal/Warning/Critical) backed by colors.
- Time-series panels use simple PromQL: `megaraid_drive_temperature_celsius` for per-drive lines, with legend `{{ enclosure }}:{{ slot }}`.

## Implementation notes

The JSON file size makes inline review tedious. Build the dashboard in the Grafana UI first, export via "JSON Model" tab, paste into the file. Then verify file-based provisioning loads it (PR-049 watches the dir at 30 s intervals).

UID `megaraid-overview` is stable. Bookmark links in operator notes use `/d/megaraid-overview` and survive future renames.

Stat panels with 0/1/2 -> Optimal/Warning/Critical mappings reuse the encoding chosen in PR-044/045. If the encoding ever changes, update both.

Keep the panel grid TIGHT: 24-column grid, all panels sized to fit on a 1080p monitor without scrolling. The §3 "one screen" principle applies to Grafana too.

Time-series colors: rely on the default Grafana palette. Do not hardcode per-drive colors; new drives after replacement get a new color automatically.

## Success criteria

1. `grafana/dashboards/overview.json` parses as valid JSON.
2. After `docker compose restart grafana`, the dashboard appears in Grafana's Browse view under "Megaraid Overview".
3. All 8 panels render without "no data" if metrics are flowing.
4. Threshold colors (green/yellow/red) apply correctly on Stat panels.
5. Time-series panels show per-drive temperature lines with `{e}:{s}` labels.
6. Refresh every 30 s.
7. README documents the dashboard URL.
8. CI passes (jsonlint or yamllint as configured).
9. No file outside the explicit lists is modified.
