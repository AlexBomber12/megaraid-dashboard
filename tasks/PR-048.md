## Problem

PR-047 bootstraps the empty repo. Prometheus is up but has no scrape targets and no retention configured beyond compose flags. This PR adds the actual `prometheus.yml` with megaraid-dashboard as a target, defines 90-day retention, and reserves a placeholder for the future sms-relay.

- Job target: `192.168.50.2:8091` (the NAS LAN IP plus the metrics port from PR-043).

Out of scope:

## Implementation notes

The 15-second scrape interval matches Prometheus convention. Megaraid-dashboard's collector runs every 300 s, so 15-second scrapes mostly return cached data (PR-044's MegaraidCollector already caches by snapshot id); the scrape cost is low.

Retention 90 days fits comfortably on the NAS (Prometheus TSDB compacts well). PR-052 trend dashboards rely on the 90-day window for monthly views.

Adding a scrape target later: edit `prometheus.yml`, `docker compose exec prometheus kill -HUP 1` to reload without restart, or `docker compose restart prometheus`.
