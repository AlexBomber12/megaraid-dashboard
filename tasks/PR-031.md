# PR-031: install.sh: sudoers + systemd + final smoke

## Problem

Final installer stage. After PR-030 the env file is configured, but the system still needs: a sudoers fragment that lets the megaraid user invoke storcli64, a systemd unit, the alembic upgrade run via the preflight script (PR-012), and a final web-port smoke test (curl /healthz). This PR closes the loop and ships `scripts/uninstall.sh` for symmetry.

## Scope

In scope:
- `phase_sudoers`: write `/etc/sudoers.d/megaraid-dashboard` with `megaraid ALL=(root) NOPASSWD: /usr/local/sbin/storcli64`, mode 0440 (sudoers requires this), validate with `visudo -c -f`.
- `phase_systemd`: copy `deploy/megaraid-dashboard.service` to `/etc/systemd/system/`, run `systemctl daemon-reload`, enable + start the unit.
- `phase_journald`: copy `deploy/journald-megaraid.conf` (PR-017) to the right namespaced drop-in directory.
- `phase_finalize`: wait up to 30 s for the unit to become `active`; curl `http://127.0.0.1:8090/healthz`; print summary block (URL, log command, edit-config command).
- New `scripts/uninstall.sh`: stops + disables the unit, removes `/etc/systemd/system/megaraid-dashboard.service`, removes sudoers, removes journald drop-in. Does NOT remove `${INSTALL_PREFIX}` or `${DATA_DIR}` by default; `--purge` flag does.
- Tests: shellcheck on both scripts; visudo dry-run check on the sudoers fragment generation.

Out of scope:
- Backup script for `${DATA_DIR}/megaraid.db`. That is a manual runbook item from §6.
- Health probe over HTTPS via nginx. PR-013 (nginx docs) handles the proxy; the installer's smoke is plain HTTP on 127.0.0.1.
- Auto-update mechanism. The operator runs `scripts/install.sh` again after `git pull`.

## Files to modify

`scripts/install.sh`:
- Append phase functions:

```bash
phase_sudoers() {
  log_info "Phase 8: sudoers"
  local sudoers="/etc/sudoers.d/megaraid-dashboard"
  cat > "${sudoers}.tmp" <<EOF
${INSTALL_USER} ALL=(root) NOPASSWD: ${STORCLI_PATH}
EOF
  chmod 0440 "${sudoers}.tmp"
  if visudo -c -f "${sudoers}.tmp" >/dev/null 2>&1; then
    mv "${sudoers}.tmp" "${sudoers}"
    log_info "wrote ${sudoers}"
  else
    rm -f "${sudoers}.tmp"
    log_fail "sudoers fragment failed visudo check"
  fi
}

phase_systemd() {
  log_info "Phase 9: systemd unit"
  install -m 0644 -o root -g root \
    "${INSTALL_PREFIX}/src/deploy/megaraid-dashboard.service" \
    /etc/systemd/system/megaraid-dashboard.service
  systemctl daemon-reload
  systemctl enable megaraid-dashboard.service
}

phase_journald() {
  log_info "Phase 10: journald drop-in"
  install -d -m 0755 /etc/systemd/journald@megaraid-dashboard.conf.d
  install -m 0644 -o root -g root \
    "${INSTALL_PREFIX}/src/deploy/journald-megaraid.conf" \
    /etc/systemd/journald@megaraid-dashboard.conf.d/00-retention.conf
  systemctl restart "systemd-journald@megaraid-dashboard.service" 2>/dev/null || true
}

phase_finalize() {
  log_info "Phase 11: start + smoke"
  systemctl restart megaraid-dashboard.service
  local i
  for i in {1..30}; do
    if systemctl is-active --quiet megaraid-dashboard.service; then
      break
    fi
    sleep 1
  done
  systemctl is-active --quiet megaraid-dashboard.service || log_fail "service failed to become active"

  local healthz
  healthz="$(curl -fs http://127.0.0.1:${APP_PORT}/healthz || true)"
  if [[ -z "${healthz}" ]]; then
    log_fail "healthz did not respond"
  fi
  log_info "healthz: ${healthz}"

  cat <<SUMMARY

==============================
INSTALL COMPLETE
==============================
Service:   megaraid-dashboard.service
Status:    $(systemctl is-active megaraid-dashboard.service)
URL:       http://$(hostname -f):${APP_PORT}/  (basic auth)
Healthz:   http://$(hostname -f):${APP_PORT}/healthz
Logs:      journalctl --namespace=megaraid-dashboard -f
Config:    ${ENV_FILE}
Reload:    sudo systemctl restart megaraid-dashboard.service
Uninstall: sudo bash ${INSTALL_PREFIX}/src/scripts/uninstall.sh

Reverse proxy: see ${INSTALL_PREFIX}/src/docs/PROXY-SETUP.md
==============================
SUMMARY
}
```

## Files to create

`scripts/uninstall.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

PURGE="false"
for arg in "$@"; do
  case "${arg}" in
    --purge) PURGE="true" ;;
  esac
done

INSTALL_USER="megaraid"
INSTALL_PREFIX="/opt/megaraid-dashboard"
DATA_DIR="/var/lib/megaraid-dashboard"
ETC_DIR="/etc/megaraid-dashboard"

[[ ${EUID} -eq 0 ]] || { echo "must run as root" >&2; exit 1; }

systemctl stop megaraid-dashboard.service 2>/dev/null || true
systemctl disable megaraid-dashboard.service 2>/dev/null || true
rm -f /etc/systemd/system/megaraid-dashboard.service
rm -f /etc/sudoers.d/megaraid-dashboard
rm -f /etc/systemd/journald@megaraid-dashboard.conf.d/00-retention.conf
systemctl daemon-reload

if [[ "${PURGE}" == "true" ]]; then
  rm -rf "${INSTALL_PREFIX}" "${DATA_DIR}" "${ETC_DIR}"
  userdel "${INSTALL_USER}" 2>/dev/null || true
  echo "purged"
else
  echo "service removed; data and config preserved at ${DATA_DIR} and ${ETC_DIR}"
fi
```

## Implementation notes

The sudoers fragment passes through `visudo -c -f` before being placed under `/etc/sudoers.d/`. A malformed sudoers file can lock root out of sudo on the next `sudo` call; validation is mandatory.

The systemd unit installation copies the file from the freshly rsynced source tree at `${INSTALL_PREFIX}/src/deploy/`. This guarantees the deployed unit matches the source revision the operator just installed, not whatever stale copy might be on disk.

The finalize phase waits up to 30 seconds for the service to become active. The wait covers: alembic upgrade head (PR-012 preflight), uvicorn boot, first DB connection. On a slow disk the boot can take a few seconds; 30 s is generous without being indefinitely patient.

The `curl /healthz` call uses the loopback interface and bypasses any future nginx proxy. This isolates the installer's success criterion to "the service itself is up", independent of whether the operator has yet wired the proxy.

The summary block at the end is the operator's first reference card. Print it boldly so it does not scroll past unnoticed in a long install log.

`uninstall.sh` is conservative by default: data and config are preserved. `--purge` is the explicit "wipe everything" flag. The PR description should mention this contract.

If sudoers validation fails (return code from `visudo -c -f`), abort cleanly. Do NOT leave a half-written `.tmp` file behind; the script removes it on the failure branch.
